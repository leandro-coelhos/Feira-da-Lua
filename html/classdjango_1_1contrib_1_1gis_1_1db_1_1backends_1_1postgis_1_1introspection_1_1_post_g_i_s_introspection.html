django.contrib.gis.db.backends.postgis.introspection.PostGISIntrospection.get_field_type ( self,
data_type,
description ) 
Hook for a database backend to use the cursor description to
match a Django field type to a database column.

For Oracle, the column data_type on its own is insufficient to
distinguish between a FloatField and IntegerField, for example.

Reimplemented from django.db.backends.postgresql.introspection.DatabaseIntrospection.

django.contrib.gis.db.backends.postgis.introspection.PostGISIntrospection.get_geometry_type ( self,
table_name,
description ) 
The geometry type OID used by PostGIS does not indicate the particular
type of field that a geometry column is (e.g., whether it's a
PointField or a PolygonField).  Thus, this routine queries the PostGIS
metadata tables to determine the geometry type.
